About Pocket SRE

Pocket SRE is a small chat app that helps you triage website incidents and remembers your stack and prior context per session.

How it works
  1. Your browser generates a sessionId and stores it in localStorage.
  2. Every message is sent to the Worker at POST /api/chat.
  3. The Worker routes the request to a Durable Object instance for that session.
  4. The Durable Object coordinates a multi-step flow: parse → clarify (max 2) → final runbook.
  5. It stores chat history + a small profile (stack/domain/notes) in persistent storage.
  6. Older chat is summarized to keep token usage small.
What you get
  • A prioritized runbook checklist (3–7 steps)
  • Three likely root causes with confidence
  • A short status update you can paste into Slack/email
Privacy note

Memory is stored per Durable Object session. Clear your browser storage to start fresh.
